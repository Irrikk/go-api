# DEPLOY.md

## Описание проекта

Это небольшой API-сервис на Go с PostgreSQL, содержащий несколько endpoint'ов:

- `/health`
- `/hello`
- `GET /user/{id}`
- `POST /user`

Приложение контейнеризовано с помощью Docker и автоматически деплоится на виртуальную машину через GitHub Actions.

---

## Архитектура решения

### Что где работает

- **Go API-приложение** запускается в **Docker-контейнере**
- **PostgreSQL** установлен **на хосте виртуальной машины**, а не в контейнере
- **CI/CD** реализован через **GitHub Actions**
- Для деплоя используется **self-hosted runner**, установленный прямо на ВМ

---

## Как устроен деплой

Деплой реализован автоматически через **GitHub Actions**.

После каждого `push` в ветку `main` запускается CI/CD pipeline, который выполняется на **self-hosted runner**, установленном непосредственно на виртуальной машине.

Это означает, что сборка и запуск контейнера происходят прямо на целевой ВМ, без отдельного SSH-деплоя с внешнего сервера.

---

## Этапы CI/CD pipeline

Pipeline состоит из четырёх этапов:

### 1. Lint
Проверка Go-кода:
- `gofmt`
- `go vet`

### 2. Build
Сборка Go-приложения:
- `go build`

### 3. Docker
Сборка Docker-образа приложения:
- `docker build -t go-api .`

### 4. Deploy
Автоматический деплой новой версии приложения на ВМ:
- остановка старого контейнера
- удаление старого контейнера
- запуск нового контейнера с актуальной версией приложения

---

## Команда запуска контейнера

Во время деплоя контейнер запускается следующей командой:

```bash
docker run -d \
  --name go-api \
  --restart always \
  -p 8080:8080 \
  --env-file /home/user/deploy/go-api.env \
  go-api
